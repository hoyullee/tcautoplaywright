- **콘텐츠**: GNB의 콘텐츠 탭을 클릭하면 진입하는 페이지
- **소셜**: GNB의 소셜 탭을 클릭하면 진입하는 페이지
- **프리랜서**: GNB의 프리랜서 탭을 클릭하면 새 페이지에서 진입되는 페이지

- **GNB**: Global Navigation Bar, 페이지 상단 글로벌 네비게이션 바
- **LNB**: Local Navigation Bar, 페이지 좌측 또는 서브 네비게이션 바

## 코드 작성 규칙

### 필수 구조
```python
import sys
from playwright.async_api import async_playwright
import asyncio
import os
import pytest

TEST_EMAIL = "..."
TEST_PASSWORD = "..."

@pytest.mark.asyncio
async def test_main():
    async with async_playwright() as p:
        browser = await p.chromium.launch(headless=True)
        context = await browser.new_context(
            locale='ko-KR',
            timezone_id='Asia/Seoul'
        )
        page = await context.new_page()

        try:
            os.makedirs('screenshots', exist_ok=True)
            await page.goto('URL', timeout=30000)
            await page.wait_for_load_state('networkidle')

            # 테스트 로직

            await page.screenshot(path='screenshots/test_{NO}_success.png  # {NO}는 두 자리 숫자 (예: 01, 02, 10)')
            print("AUTOMATION_SUCCESS")
            return True

        except Exception as e:
            await page.screenshot(path='screenshots/test_{NO}_failed.png   # {NO}는 두 자리 숫자 (예: 01, 02, 10)')
            print(f"AUTOMATION_FAILED: {e}")
            raise  # pytest가 FAILED로 인식하도록 반드시 re-raise

        finally:
            await browser.close()

if __name__ == "__main__":
    result = asyncio.run(test_main())
    sys.exit(0 if result else 1)
```

### hover로만 노출되는 버튼 처리 패턴
마우스 hover 시에만 나타나는 버튼(예: 카드 위의 더보기/3점 버튼)은 아래 순서로 처리:
```python
# 1. Playwright hover 사용 (mouse.move 대신)
await target_element.hover()
await page.wait_for_timeout(500)

# 2. hover 후 버튼이 visible 상태가 될 때까지 대기
btn = target_element.locator('[aria-label="더보기"]')
await btn.wait_for(state='visible', timeout=5000)
await btn.click()

# 3. 위 방법 실패 시 JS로 강제 클릭 (hover 없이도 동작)
await page.evaluate("""(el) => {
    el.dispatchEvent(new MouseEvent('mouseover', {bubbles: true}));
    el.dispatchEvent(new MouseEvent('mouseenter', {bubbles: true}));
}""", await target_element.element_handle())
await page.wait_for_timeout(500)
await btn.click(force=True)
```

### page.evaluate() 성능 규칙

> ⚠️ 이 규칙은 **`page.evaluate()` 내부 JS 코드**에만 적용됩니다.
> Playwright 로케이터(`page.locator('[class*="..."]')`)는 해당 없음.

`page.evaluate()` 내부의 `querySelectorAll`에서 `[class*="..."]` 셀렉터는 DOM 전체를 순회하므로 **가능하면 시맨틱 셀렉터로 대체**하세요.

#### 셀렉터 대체 우선순위 (evaluate 내부)
| 상황 | 느린 방식 | 빠른 대체 |
|------|-----------|-----------|
| 체크박스 탐색 | `[class*="check"]` | `input[type="checkbox"], [role="checkbox"]` |
| 다이얼로그/모달 탐색 | `[class*="modal"]` | `[role="dialog"]` |
| 버튼 탐색 | `[class*="button"]` | `button, [role="button"]` |

> ⚠️ `[class*="overlay"]`는 로딩 스피너·배경 딤 등 다이얼로그가 아닌 요소에도 사용되므로 `[role="dialog"]`로 대체하지 말 것. 오버레이를 찾아야 한다면 목적에 맞는 구체적 셀렉터를 사용하세요.
> 시맨틱 속성이 없는 요소는 `class*=` 사용을 허용하되, `.slice(0, N)`으로 처리 수를 제한하세요.

#### getBoundingClientRect() 규칙
`getBoundingClientRect()`는 호출마다 레이아웃 재계산(reflow)을 강제하므로:
```javascript
// 🚫 지양 — 매칭 요소 전체에 getBoundingClientRect 호출
[...document.querySelectorAll('button')].map(el => {
    const rect = el.getBoundingClientRect();
    return { visible: rect.width > 0 };
});

// ✅ 권장 — slice로 상한 제한 후 처리
[...document.querySelectorAll('button')].slice(0, 30).map(el => {
    const rect = el.getBoundingClientRect();
    return { visible: rect.width > 0 };
});
```

**규칙 요약:**
- `querySelectorAll` + `.map()` + `getBoundingClientRect()` 조합 시 반드시 `.slice(0, N)` 선행 (체크박스: 20개, 버튼: 30개 권장)
- 시맨틱 속성(`role`, `type`, `tag`)이 있는 경우 `class*=` 대신 그것을 우선 사용
