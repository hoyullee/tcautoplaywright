### 용어 설명
- **채용 홈**: 원티드 메인 채용 페이지
- **탐색 페이지**: GNB의 채용 탭을 클릭하면 진입하는 페이지
- **이력서 페이지**: GNB의 이력서 탭을 클릭하면 진입하는 페이지 (로그인 여부에 따라 URL 다름)
- **교육이벤트**: GNB의 교육•이벤트 탭을 클릭하면 진입하는 페이지
- **콘텐츠**: GNB의 콘텐츠 탭을 클릭하면 진입하는 페이지
- **소셜**: GNB의 소셜 탭을 클릭하면 진입하는 페이지
- **프리랜서**: GNB의 프리랜서 탭을 클릭하면 새 페이지에서 진입되는 페이지
- **포지션 상세 페이지**: 탐색 페이지에서 포지션 카드를 선택하면 진입되는 페이지 (URL 패턴: `https://www.wanted.co.kr/wd/{포지션ID}`)
- **GNB**: Global Navigation Bar, 페이지 상단 글로벌 네비게이션 바
- **LNB**: Local Navigation Bar, 페이지 좌측 또는 서브 네비게이션 바

## 코드 작성 규칙

### 필수 구조
```python
import sys
from playwright.async_api import async_playwright
import asyncio
import os
import pytest

TEST_EMAIL = "..."
TEST_PASSWORD = "..."

@pytest.mark.asyncio
async def test_main():
    async with async_playwright() as p:
        browser = await p.chromium.launch(headless=True)
        context = await browser.new_context(
            locale='ko-KR',
            timezone_id='Asia/Seoul'
        )
        page = await context.new_page()

        try:
            os.makedirs('screenshots', exist_ok=True)
            await page.goto('URL', timeout=30000)
            await page.wait_for_load_state('networkidle')

            # 테스트 로직

            await page.screenshot(path='screenshots/test_{NO}_success.png  # {NO}는 두 자리 숫자 (예: 01, 02, 10)')
            print("AUTOMATION_SUCCESS")
            return True

        except Exception as e:
            await page.screenshot(path='screenshots/test_{NO}_failed.png   # {NO}는 두 자리 숫자 (예: 01, 02, 10)')
            print(f"AUTOMATION_FAILED: {e}")
            return False

        finally:
            await browser.close()

if __name__ == "__main__":
    result = asyncio.run(test_main())
    sys.exit(0 if result else 1)
```

### 로케이터 우선순위
1. `page.get_by_role('button', name='텍스트')`
2. `page.get_by_text('텍스트')`
3. `page.locator('css-selector')`

### 스크린샷 규칙 (반드시 준수)
- **TC당 스크린샷은 1장만** 허용
- 성공 시: `screenshots/test_{NO}_success.png  # {NO}는 두 자리 숫자 (예: 01, 02, 10)` 1장만 저장
- 실패 시: `screenshots/test_{NO}_failed.png   # {NO}는 두 자리 숫자 (예: 01, 02, 10)` 1장만 저장
- 중간 단계(페이지 로드, 버튼 클릭 등) 스크린샷 절대 금지
- `page.screenshot()`은 try 블록 마지막(성공 확정 후)과 except 블록(실패 확정 후) 각각 1회씩만 호출

### 절대 금지
- `sys.stdout = io.TextIOWrapper(...)` 사용 금지 — pytest stdout 캡처를 파괴함
- 중간 단계 스크린샷 추가 금지 (step1, step2, login_page 등 이름의 스크린샷)

## 작업 절차

1. `test/test_{NO}_working.py` 파일 생성 후 코드 작성
2. `python test/test_{NO}_working.py` 실행
3. 성공 시 → `test/test_{NO}_success.py`로 이름 변경
4. 실패 시 → 에러 분석 후 코드 수정하여 재시도
5. 반드시 마지막에 `AUTOMATION_SUCCESS` 또는 `AUTOMATION_FAILED: 에러메시지` 출력
